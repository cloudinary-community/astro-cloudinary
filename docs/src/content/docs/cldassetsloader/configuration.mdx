### `context`

Whether to include Contextual Metadata (`context`) in the resource object.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
context: true
```
</CodeBlock>

[Learn more about Contextual Metadata](https://cloudinary.com/documentation/contextual_metadata) in the Cloudinary docs.

#### `deliveryType`

The delivery type of the assets you want to query.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
deliveryType: 'private'
```
</CodeBlock>

[Learn more about Delivery Types](https://cloudinary.com/documentation/upload_parameters#delivery_types) on the Cloudinary docs.

#### `fields`

The asset fields to include in the resource object response.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
fields: ['width', 'height', 'context']
```
</CodeBlock>

#### `folder`

The folder the assets should be queried from.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
folder: 'samples/landscapes'
```
</CodeBlock>

#### `limit`

The maximum amount of results to query.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
limit: 20
```
</CodeBlock>

### `metadata`

Whether to include Structured Metadata (`metadata`) in the resource object.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
metadata: true
```
</CodeBlock>

[Learn more about Structured Metadata](https://cloudinary.com/documentation/structured_metadata) in the Cloudinary docs.

### `moderation`

Whether to include the moderation status (`moderation`) in the resource object.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
moderation: true
```
</CodeBlock>

[Learn more about Moderation on Upload](https://cloudinary.com/documentation/upload_parameters#moderation) in the Cloudinary docs.

#### `recursive`

Whether to include assets from subfolders when querying a folder. When set to `true`, the loader will search through all subfolders of the specified folder.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
// Include assets from subfolders
recursive: true
```
</CodeBlock>

**Note:** Recursive folder queries work differently depending on your Cloudinary folder mode:
- **Dynamic folders**: Discovers and queries each subfolder individually
- **Fixed folders**: Uses prefix matching which naturally includes subfolders

#### `resourceType`

The type of the assets you want to query.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
resourceType: 'video'
```
</CodeBlock>

[Learn more about Asset Types](https://cloudinary.com/documentation/upload_parameters#asset_types) on the Cloudinary docs.

### `tags`

Whether to include the asset's tags (`tags`) in the resource object.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
tags: true
```
</CodeBlock>

[Learn more about Tags](https://cloudinary.com/documentation/tags) in the Cloudinary docs.

#### `type`

The type of query to make for assets, including list or search.

This primarily differs in how the requests are made, using the Admin API or the Search API.

**Examples**

<CodeBlock alwaysExpanded>
```jsx
type: 'search'
```
</CodeBlock>
